# requirements.md - 要件定義

## Goal（何を作るか）

ホッキョクグマの環境保護につながるWEBサイトをNext.jsで作成する。

## Must（必須機能）

- [x] Next.js 14を使用したモダンなWEBサイト
- [x] ホッキョクグマの現状・課題を伝えるコンテンツ
- [x] 環境保護への行動を促すUI
- [v] レスポンシブデザイン（モバイル対応）
- [v] 基本的なナビゲーション

## Nice（あったら良い機能）

- [ ] 寄付・支援への導線
- [ ] ニュース・更新情報セクション
- [ ] 統計データの可視化
- [ ] 多言語対応
- [ ] アクセシビリティ対応

## Out of scope（対象外）

- ユーザー認証・ログイン機能
- データベース連携（初期段階）
- 決済機能（MVP段階）
- 管理画面

## 目的（Message）

- 主メッセージ：ホッキョクグマを守るために、いま知ることから始めよう
- ゴール：読者が「現状を理解」し、「自分の行動を1つ選べる」状態にする

## ターゲット（Audience）

- 主対象：気候変動に詳しくない一般層（忙しい／知識が薄い）
- 副対象：関心が深まり始めた層（ニュースで追う程度）
- 想定読了：3分で「海氷＝狩りの足場」「地域差がある」まで説明できる

## トーン / ライティング方針（Tone）

- 入口はやさしく、読めば少し玄人感（**二層構造**）
- 断定で煽らない／不安より「理解できた」を優先
- 文章は短く、1文1情報。専門語は最小限

## 専門性の出し方（Expertise）

- 各セクションの専門ワードは **1語（最大2語）**
- 専門語は出した直後に言い換える（例：個体群＝地域ごとのグループ）
- 必ず「地域差」「推定の幅（不確実性）」に触れ、過度な一般化を避ける

## 情報設計（IA）

- 構成：Hero → About → Threat → Data → Importance → Action → CTA
- 導線：**知る（現状）→ 理解する（データ/背景）→ 動く（アクション）**
- CTAは常に2択：**「現状を知る」/「できることを見る」**

## 連携方針（Future）

- Polar Bears International 等の専門組織との連携は「将来的に目指す」と表現（現時点で断定しない）

## Done条件（受け入れ条件）

1. ローカル環境で`npm run dev`で起動できる
2. ホッキョクグマ保護に関する主要コンテンツが表示される
3. モバイル・デスクトップで適切に表示される
4. 基本的なナビゲーションが機能する
